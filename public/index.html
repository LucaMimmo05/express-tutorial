General & Utilities

Base
GET /

Home page e documentazione.

GET /fetch

Visualizza i cookie inviati dal client.

POST /form

Upload di file e dati.

BODY (multipart/form-data):
image: <file>
...altri campi form opzionali
GET /dashboard

Endpoint protetto.

HEADERS:
Authorization: Bearer <token_jwt>

Users

Auth & Search
POST /user/register

Registrazione nuovo utente.

BODY (JSON):
{
"username": "string",
"password": "string"
}
POST /user/login

Login utente. Restituisce un token JWT.

BODY (JSON):
{
"username": "string",
"password": "string"
}
GET /user/search

Ricerca utenti.

QUERY PARAMS:
?keyword=<string>
GET /user/:username

Dettagli utente specifico.

URL PARAMS:
:username (string) - Nome utente da cercare

Persons

MongoDB CRUD
POST /person

Crea una nuova persona.

BODY (JSON):
{
"name": "string", *required
"email": "string", *required
"age": number *required
}
PUT /person

Aggiorna una persona esistente.

BODY (JSON):
{
"id": "string", *required
"name": "string",
"email": "string",
"age": number
}
GET /person/:id

Ottieni persona per ID.

URL PARAMS:
:id (MongoDB ObjectId)

Products

Static Data
POST /product

Crea un nuovo prodotto (in memoria).

BODY (JSON):
{
"name": "string",
"price": number
}
GET /product/:id

Ottieni prodotto per ID numerico.

URL PARAMS:
:id (number)
